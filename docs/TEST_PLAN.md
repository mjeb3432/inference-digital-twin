## Key Interactions to Verify
- Submitting a valid scenario enqueues compute and returns a run id on /explorer.
- Rapid successive scenario edits on /explorer show latest run only (no stale overwrite).
- Viewing /runs/:id transitions cleanly across queued, running, success, failure.
- Opening /reports/:id/provenance shows scenario hash, artifact ids, and module versions.

## Edge Cases
- Queue unavailable on submit shows retryable actionable error on /explorer.
- Unsupported module/runtime version is rejected with clear validation messaging.
- Long-running run crosses TTL and transitions to timeout with user-visible guidance.
- Two equivalent scenario JSON payloads (different key order) generate identical hash and cache hit.
- Exported scenario+report rerun reproduces outputs within declared tolerance.

## Critical Paths
- Author scenario -> enqueue async run -> worker computes modules -> prediction report persists -> report + provenance render.
- Calibration artifact update -> compatibility check -> new runs pick correct artifact version.
- Regression guard -> golden scenarios execute in CI -> PR fails on unexplained drift beyond tolerance.
