## Purpose

This page explains where key formulas and assumptions live so public users can audit model behavior.

## Reference Categories

### Infrastructure and Economics

- site capacity envelope assumptions
- permit cost/timeline assumptions
- power blend and capex assumptions
- capex and opex component rollups

### Performance and Network

- latency decomposition assumptions
- topology and scale penalties
- throughput and concurrency caps
- MFU and utilization assumptions

### Reliability and Operations

- health score components
- uptime projection inputs
- monitoring and maintenance weighting

### Calibration and Provenance

- coefficient artifacts and versions
- module versions
- scenario hashing and cache keys
- report provenance fields

## Assumption Governance

Use the Assumptions Register artifact to track:

- owner
- confidence
- evidence source
- review cadence
- downstream decision impact

## Recommended Public Contribution Pattern

When proposing assumption changes, include:

1. assumption ID and current value
2. proposed value and rationale
3. expected metric impact
4. evidence source
5. validation plan
