# Roadmap and Contribution Guide

## Roadmap Themes

### Near-Term

- stronger interconnect fidelity
- richer calibration coverage
- improved reproducibility bundle workflow

### Mid-Term

- broader hardware/runtime support
- deeper scenario benchmarking sets
- structured publication workflow for research comparisons

### Long-Term

- open ecosystem for shared scenario packs
- peer-review style assumption update process
- deeper integration with real validation datasets

## How to Contribute

1. Identify problem or opportunity
2. Tie contribution to a specific assumption, module behavior, or workflow gap
3. Provide evidence and expected impact
4. Include reproducible before/after scenario comparisons

## Contribution Standards

- Keep assumptions explicit
- Keep comparisons controlled
- Preserve deterministic behavior
- Document limitations honestly

## Suggested Public Contribution Types

- assumption review and evidence updates
- calibration artifact proposals
- scenario pack contributions
- docs and user-guide improvements
- validation checklists and reproducibility tooling
